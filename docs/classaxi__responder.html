task axi_responder::read_data ( )

Read Data channel thread.

  1. Deassert rvalid
  2. wait for tlm packet (responder packet from axi_monitor)
  3. loop through data
  4. if ready and valid, wait X clocks where x>=0, then check for any more queued items
  5. if avail, then fetch and goto 'Loop through data' step.
  6. if no items to be driven on next clk, drive all read data signals low and goto 'Wait for TLM item in mailbox' step.

task axi_responder::run_phase ( uvm_phase  phase)

Launches channel responder threads and then acts as a dispatcher.

After launching 5 different threads (one for each channel), this task acts as a dispatcher. It waits for TLM packets and then stuffs them into the appropriate thread's mailbox. IE: If it's an AXI write packet then it puts the packet into the write_address's mailbox so it can handle it. It the waits for the next TLM packet. NOTE: it does not wait for the other thread to finish processing the packet, it just puts it in the mailbox and then immediately waits for the next packet.

task axi_responder::write_response ( )

Write Response channel thread.

  1. Deassert bvalid
  2. Wait for TLM item in mailbox
  3. Initialize variables
  4. Write out
  5. if ready and valid, wait X clocks where x>=0, then check for any more queued items
  6. if avail, then fetch and goto 'Initialize variables' step.
  7. if no items to be driven on next clk, drive all write response signals low and goto 'Wait for TLM item in mailbox' step. \todo: response values are hardcoded. Get from response seq?
